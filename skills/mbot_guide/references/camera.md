# CameraGetTool 摄像头状态与图像获取工具业务逻辑文档

## 1. 概述
CameraGetTool 是用于获取 Mbot 机器人摄像头状态及实时图像的 smart agent 工具。它基于 HTTP Web 接口实现，无需 ROS 环境依赖，支持查询摄像头运行状态（FPS、分辨率、连接状态）、健康检查以及获取实时画面。获取的画面会自动保存到本地临时目录，并返回文件路径供多模态模型处理或展示。

## 2. 核心功能
- ✅ **状态查询**：获取摄像头实时 FPS、分辨率、连接状态及是否有画面帧。
- ✅ **健康检查**：快速判断摄像头是否正常运行且正在采集画面。
- ✅ **图像获取**：下载当前摄像头实时画面，保存至本地 TEMP_PATH 目录，并返回绝对路径。
- ✅ **多模态支持**：在获取图像时，支持传入 `model_text_input` 参数，用于指定针对该图像的多模态处理指令（如“描述图片内容”）该参数由大模型自己决定使用什么样子的文本来进行处理。
- ✅ **资源管理**：自动管理 HTTP Session，支持超时控制和异常捕获。

## 3. 输入参数说明

所有请求均通过 execute 方法传入关键字参数。

| 参数名 | 类型 | 必填 | 说明 | 约束/枚举 |
| :--- | :--- | :--- | :--- | :--- |
| `action` | string | **是** | 执行的动作类型 | `get_status` (获取完整状态)<br>`check_health` (健康检查)<br>`get_image` (获取实时照片)<br>get_camera_status (简略状态，兼容旧版) |
| `model_text_input` | string | 否* | 多模态处理指令 | 仅当 `action` 为 `get_image` 时有效。由大模型自己决定使用什么样子的文本来进行处理。 |

> **注意**：
> 1. `get_status`, `check_health`, get_camera_status 不需要 `model_text_input` 参数。
> 2. `get_image` 动作会触发图片下载和本地保存操作，耗时相对较长。

## 4. 动作详解

### 4.1 获取完整状态 (`get_status`)
查询摄像头的详细运行指标。
- **输入**: `{"action": "get_status"}`
- **后端接口**: GET `/status`
- **返回数据字段**:
  - fps: 实时帧率 (浮点数)
  - `resolution`: 画面分辨率 (字符串, 如 "640x480")
  - `status`: 连接状态 (字符串, 如 "connected")
  - `has_frame`: 是否有画面帧 (布尔值)
- **输出示例**:
  ```text
  📷 摄像头实时状态：
  🎬 实时FPS：30.5
  📐 画面分辨率：640x480
  🔌 连接状态：connected
  🖼️ 画面状态：正常采集
  ```
- **异常处理**: 如果请求失败，返回：“❌ 获取摄像头状态失败，请检查相机服务是否运行”

### 4.2 健康检查 (`check_health`)
快速验证摄像头是否可用。
- **输入**: `{"action": "check_health"}`
- **逻辑**:
  1. 调用 `/status` 接口。
  2. 检查 `has_frame` 是否为 `True`。
- **输出示例 (正常)**:
  ```text
  ✅ 摄像头运行正常 | FPS:30.5 | 状态:connected
  ```
- **输出示例 (异常)**:
  ```text
  ❌ 摄像头异常：未采集到画面或服务未启动
  ```

### 4.3 获取实时照片 (`get_image`)
下载当前帧并保存到本地，用于多模态分析。
- **输入**: 
  ```json
  {
    "action": "get_image",
    "model_text_input": ""
  }
  ```
- **逻辑**:
  1. 获取摄像头状态，确认 `has_frame` 为 `True`。
  2. 构建图片 URL: `http://{host}:{port}/get_image`。
  3. 发送 GET 请求下载图片二进制数据,并由大模型自己决定发送什么样的文本来填充`model_text_input`参数
  4. 生成唯一文件名: `camera_{uuid}_{timestamp}.jpg`。
  5. 保存图片到环境变量 TEMP_PATH 指定的目录。
  6. 返回包含图片路径的内容，以便 Agent 进行多模态处理。
- **输出行为**:
  - 成功时：返回 to_content(text=model_text_input, image=image_url)。Agent 将利用此路径加载图片并进行多模态推理。
  - 失败时：返回错误文本，如 “❌ 获取摄像头照片失败：相机未运行” 或 “❌ 照片保存失败，temp目录无写入权限”。

### 4.4 简略状态 (get_camera_status)
兼容旧版的简短状态查询。
- **输入**: `{"action": "get_camera_status"}`
- **输出示例**:
  ```text
  FPS:30.5 | 状态:connected
  ```

## 5. 业务约束与校验

1. **动作合法性**：
   - `action` 必须为 `get_status`, `check_health`, `get_image`, 或 `get_camera_status`。
   - 传入其他值将返回错误提示。

2. **网络依赖**：
   - 依赖环境变量 CAMERA_URL 和 CAMERA_PORT 配置摄像头服务地址。
   - 默认连接超时 3 秒，读取超时 5 秒。
   - 请确保摄像头 Web 服务已启动且网络可达。

3. **文件系统依赖**：
   - 依赖环境变量 TEMP_PATH 指定图片保存目录。
   - 工具会自动创建该目录（如果不存在）。
   - 确保运行进程对该目录具有写入权限。

4. **画面有效性**：
   - 只有当摄像头状态显示 `has_frame` 为 `True` 时，`get_image` 才会尝试下载图片。
   - 如果摄像头未启动或无画面，直接返回失败信息，避免无效下载。

## 7. 异常处理

工具执行失败时，会返回包含错误信息的文本内容，常见错误如下：

| 错误类型 | 返回信息示例 | 原因 |
| :--- | :--- | :--- |
| 不支持动作 | `❌ 不支持的动作：xxx，请使用 get_status / check_health / get_image` | `action` 值错误 |
| 服务异常 | `❌ 获取摄像头状态失败，请检查相机服务是否运行` | 网络超时、连接拒绝或服务返回非 200 状态码 |
| 无画面 | `❌ 获取摄像头照片失败：无画面` | 摄像头已连接但未传输视频帧 |
| 服务未启动 | `❌ 获取摄像头照片失败：相机未运行` | 无法获取摄像头状态或状态显示未运行 |
| 保存失败 | `❌ 照片保存失败，temp目录无写入权限` | 磁盘空间不足或权限问题 |
