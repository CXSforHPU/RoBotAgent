# 机械臂控制工具（arm_control）Skill工作逻辑文档
## 一、工具概述
`arm_control` 是面向智能体的机械臂控制工具，基于无ROS依赖的纯Python客户端（`MbotWebClient`）实现，支持机械臂状态获取、单个关节角度设置（指定运动时间）、夹爪角度设置（指定运动时间）三大核心功能，所有交互通过HTTP接口与机械臂服务端通信。

## 二、核心工作流程
### 2.1 工具初始化
1. 工具启动时，`ArmControlTool` 类初始化，自动读取环境配置中的 `MBOT_MOTION_URL`（机械臂服务地址）和 `MBOT_MOTTION_PORT`（服务端口）。
2. 实例化 `MbotWebClient` 客户端，创建HTTP会话（`requests.Session`），并设置会话超时时间为3秒，确保接口调用的稳定性。

### 2.2 参数校验规则
工具接收参数后，先按动作类型校验必填项和参数合法性，规则如下：

| 动作类型       | 必填参数                | 参数校验规则                                                                 |
|----------------|-------------------------|------------------------------------------------------------------------------|
| `get_status`   | `action`                | 无额外校验                                                                   |
| `set_joint`    | `action`/`joint_name`/`angle`/`time` | 1. `joint_name` 必须是 joint1/joint2/joint3/joint4/joint5<br>2. `angle` 范围 -125~125<br>3. `time` 必须是大于0的数字（秒） |
| `set_gripper`  | `action`/`angle`/`time` | 1. `angle` 范围 -125~125<br>2. `time` 必须是大于0的数字（秒）                |

### 2.3 动作执行逻辑
#### 2.3.1 获取机械臂状态（`get_status`）
1. 客户端调用 `get_arm_status()` 方法，向 `http://{host}:{port}/arm/get_status` 发送GET请求。
2. 若请求成功且返回码为200，解析返回的关节（joint1-joint5）和夹爪角度数据；若请求失败（如服务未运行、网络异常），返回失败提示。
3. 将状态数据格式化输出为自然语言文本，供智能体返回给用户。

#### 2.3.2 设置单个关节角度（`set_joint`）
1. 校验参数合法性（缺失必填项/时间≤0则直接返回参数错误）。
2. 客户端调用 `set_joint()` 方法，向 `http://{host}:{port}/arm/set_joint` 发送POST请求，携带 `joint_name` 和 `angle` 参数。
3. 发送请求后，客户端休眠指定的 `time` 秒（匹配机械臂实际运动时长）。
4. 根据接口返回码（200为成功），返回“设置成功/失败”的格式化文本。

#### 2.3.3 设置夹爪角度（`set_gripper`）
1. 校验参数合法性（缺失必填项/时间≤0则直接返回参数错误）。
2. 客户端调用 `set_gripper()` 方法，向 `http://{host}:{port}/arm/set_gripper` 发送POST请求，携带 `angle` 和 `time` 参数。
3. 发送请求后，客户端休眠指定的 `time` 秒（匹配夹爪实际运动时长）。
4. 根据接口返回码（200为成功），返回“设置成功/失败”的格式化文本。

### 2.4 异常处理机制
1. **网络异常**：所有HTTP请求捕获 `RequestException` 异常（如连接超时、服务不可达），统一返回操作失败结果。
2. **参数异常**：针对缺失必填项、参数格式错误（如时间≤0、角度超范围），返回明确的参数错误提示，不执行实际接口调用。
3. **服务异常**：若接口返回码非200，判定为操作失败，返回对应提示文本。

## 三、接口交互规范
### 3.1 输入参数格式（JSON）
```json
{
  "action": "get_status/set_joint/set_gripper",
  "joint_name": "joint1（仅set_joint需填）",
  "angle": 90（仅set_joint/set_gripper需填，范围-125~125）,
  "time": 2.5（仅set_joint/set_gripper需填，>0）
}
```

### 3.2 输出结果格式
所有执行结果均通过 `to_content` 方法封装为统一格式的列表（`List[Dict[str, Any]]`），内容为自然语言文本，示例如下：
- 成功获取状态：
  ```
  机械臂当前状态：
  joint1: 0°
  joint2: 30°
  joint3: 45°
  joint4: 60°
  joint5: 0°
  夹爪: 90°
  ```
- 成功设置关节：
  ```
  设置joint1=90°，时间=2.5s：成功
  ```
- 参数错误：
  ```
  参数错误：运动时间必须大于0秒
  ```

## 四、关键设计说明
1. **时间参数核心作用**：`time` 参数用于控制机械臂运动时长，客户端通过 `tm.sleep(time)` 休眠匹配运动时间，确保动作执行完成后再返回结果。
2. **无ROS依赖**：基于HTTP接口实现，脱离ROS环境依赖，适配纯Python部署场景。
3. **参数严格对齐**：工具参数定义严格遵循智能体要求，`required` 字段仅标记 `action`，其余参数按动作类型动态校验，兼顾灵活性和规范性。
4. **会话复用**：使用 `requests.Session` 复用HTTP连接，减少频繁建立连接的开销，提升接口调用效率。

## 五、常见问题及处理
| 问题现象                     | 可能原因                     | 处理方案                                   |
|------------------------------|------------------------------|--------------------------------------------|
| 获取状态失败                 | 机械臂服务未启动/地址端口错误 | 检查服务进程、核对 `MBOT_MOTION_URL/PORT` 配置 |
| 设置关节/夹爪返回失败        | 参数错误/服务端接口异常      | 校验参数（角度范围、时间>0）、查看服务端日志 |
| 工具执行超时                 | 网络延迟/服务响应慢          | 检查网络、调整客户端超时时间（当前为3秒）   |
