# SensorGetTool 传感器数据查询工具业务逻辑文档

## 1. 概述
SensorGetTool 是用于获取 Mbot 机器人传感器数据的智能体工具（Agent Tool）。它基于 HTTP Web 接口实现，无需 ROS 环境依赖，支持查询实时环境传感器数据（如电压、温湿度、气压、光照、空气质量、超声波）以及传感器节点的系统运行状态。该工具旨在为智能体提供环境感知能力，以便进行决策和控制。

## 2. 核心功能
- ✅ **实时数据查询**：获取电池电压、环境温度、湿度、大气压强、光照强度、空气质量指数及四个方向超声波障碍物状态。
- ✅ **节点状态监控**：获取传感器服务节点的运行状态、数据更新情况、订阅话题及最后更新时间。
- ✅ **无依赖集成**：纯 Python 实现，通过 HTTP 请求与后端服务通信，无 ROS 客户端库依赖。
- ✅ **标准化输出**：返回格式化的文本信息，便于大语言模型（LLM）理解和展示。

## 3. 输入参数说明

所有请求均通过 execute 方法传入关键字参数。

| 参数名 | 类型 | 必填 | 说明 | 约束/枚举 |
| :--- | :--- | :--- | :--- | :--- |
| `action` | string | **是** | 执行的动作类型 | `get_data` (获取实时传感器数据)<br>`get_status` (获取节点运行状态) |

> **注意**：
> 1. 本工具仅有一个必填参数 `action`。
> 2. 根据 `action` 的值，工具内部会自动调用相应的后端接口。

## 4. 动作详解

### 4.1 获取实时传感器数据 (`get_data`)
查询当前环境的所有传感器读数。
- **输入**: `{"action": "get_data"}`
- **后端接口**: GET `/sensor_data`
- **返回数据字段**:
  - `voltage`: 电池电压 (单位: V)
  - `temperature`: 环境温度 (单位: ℃)
  - `humidity`: 环境湿度 (单位: %RH)
  - `pressure`: 大气压强 (单位: kPa)
  - `light`: 光照强度 (单位: Lux)
  - `mp503`: 空气质量指标 1
  - `mp2`: 空气质量指标 2
  - `sonar1` - `sonar4`: 超声波传感器状态 (布尔值: True 表示检测到障碍物, False 表示无障碍物)
- **输出示例**:
  ```text
  📊 传感器实时数据：
  🔋 电池电压：12.5 V
  🌡️ 环境温度：25.3 ℃
  💧 环境湿度：45.0 %RH
  🌍 大气压强：101.3 kPa
  💡 光照强度：300 Lux
  🌫️ 空气质量1：Good
  🌫️ 空气质量2：Good
  📡 超声波1：无障碍物
  📡 超声波2：检测到障碍物
  📡 超声波3：无障碍物
  📡 超声波4：无障碍物
  ```
- **异常处理**: 如果请求失败或数据为空，返回：“❌ 获取传感器数据失败，请检查传感器服务是否运行”

### 4.2 获取节点运行状态 (`get_status`)
查询传感器服务后台节点的健康状况。
- **输入**: `{"action": "get_status"}`
- **后端接口**: GET `/sensor_status`
- **返回数据字段**:
  - `node_status`: 节点状态 (字符串: "running" 或其他)
  - `has_data`: 是否有最新数据 (布尔值)
  - `topic`: 订阅的 ROS 话题名称
  - `data_update_time`: 数据最后更新的时间戳或间隔 (单位: s)
- **输出示例**:
  ```text
  🔧 传感器节点状态：
  ✅ 节点状态：运行中
  📥 数据状态：有数据
  🌐 订阅话题：/sensor/combined_data
  ⏰ 最后更新时间：1.2s
  ```
- **异常处理**: 如果请求失败或数据为空，返回：“❌ 获取节点状态失败，请检查传感器服务是否运行”

## 5. 业务约束与校验

1. **动作合法性**：
   - `action` 必须为 `get_data` 或 `get_status`。
   - 传入其他值将返回错误提示：“❌ 不支持的动作：{action}，请使用 get_data / get_status”

2. **网络依赖**：
   - 依赖环境变量 `SENSOR_URL` 和 `SENSOR_PORT` 配置传感器服务地址。
   - 默认请求超时时间为 3 秒。
   - 请确保传感器 Web 服务已启动且网络可达。

3. **数据有效性**：
   - 工具假设后端返回的 JSON 结构中包含预期的字段（如 `voltage`, `temperature` 等）。如果后端数据结构变更，可能导致 KeyError 异常（当前代码未做详细的字段存在性校验，建议后端保证数据结构稳定）。

## 7. 异常处理

工具执行失败时，会返回包含错误信息的文本内容，常见错误如下：

| 错误类型 | 返回信息示例 | 原因 |
| :--- | :--- | :--- |
| 不支持动作 | `❌ 不支持的动作：xxx，请使用 get_data / get_status` | `action` 值错误 |
| 服务异常 | `❌ 获取传感器数据失败，请检查传感器服务是否运行` | 网络超时、连接拒绝或服务返回非 200 状态码 |
| 服务异常 | `❌ 获取节点状态失败，请检查传感器服务是否运行` | 网络超时、连接拒绝或服务返回非 200 状态码 |
